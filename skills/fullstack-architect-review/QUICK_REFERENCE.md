# 快速审查清单（一页纸速查）

## 审查优先级

**正确性 > 安全性 > 可维护性 > 性能 > 用户体验 > 简洁性**

---

## P0 必查项（安全 + 正确性）

- [ ] XSS: 搜索 `dangerouslySetInnerHTML` / `v-html` / `innerHTML` -- 是否经过 DOMPurify 清理?
- [ ] SQL注入: 搜索原始SQL拼接 / `$queryRawUnsafe` -- 是否使用参数化查询?
- [ ] 认证: 所有需保护的路由/Action 是否有认证中间件?
- [ ] 授权: 用户只能操作自己的资源? 是否校验了资源所有权?
- [ ] 密码存储: 是否使用 bcrypt(cost>=12) / argon2id? 禁止 MD5/SHA1
- [ ] 密钥: 搜索 `password=` / `secret=` / `api_key=` -- 是否有硬编码?
- [ ] .env: 是否在 .gitignore 中? 是否有 .env.example?
- [ ] eval: 搜索 `eval()` / `new Function()` / `exec()` -- 禁止使用

## P1 重点项（架构 + 质量）

### 前端
- [ ] 组件是否遵循单一职责? 超过300行的组件需拆分
- [ ] 状态是否在正确的层级? 避免不必要的全局状态
- [ ] 异步状态是否处理了 loading / error / empty 三态?
- [ ] 列表超过100项是否使用虚拟滚动?
- [ ] 路由级别是否使用代码分割?
- [ ] TypeScript strict 模式是否启用? `any` 类型数量?

### 后端
- [ ] Controller/Service/Repository 分层是否清晰?
- [ ] 所有输入是否在服务端验证? (Zod / Pydantic / class-validator)
- [ ] 是否有 N+1 查询? (使用 include/select_related)
- [ ] 是否有统一的错误处理中间件?
- [ ] 分页是否强制? 是否限制了最大 pageSize?
- [ ] API 响应格式是否统一?

### 集成
- [ ] 前后端数据结构是否一致? 是否有共享类型?
- [ ] Token 刷新是否处理了并发请求竞态?
- [ ] CORS 配置是否最小化?

## P2 优化项

- [ ] 图片: WebP/AVIF + lazy loading + 明确宽高
- [ ] 缓存: 是否有 Redis/内存缓存策略?
- [ ] Bundle: 是否有 bundle analyzer 分析?
- [ ] 日志: 是否使用结构化日志? 是否泄漏敏感信息?
- [ ] 测试: 关键业务逻辑是否有单元测试?

---

## 6维度评分速查

| 分数 | 含义 |
|------|------|
| 9-10 | 生产就绪,可作为范例 |
| 7-8 | 少量改进即可上线 |
| 5-6 | 存在短板,需改进 |
| 3-4 | 多处严重问题 |
| 0-2 | 安全漏洞/架构硬伤,禁止部署 |

维度: 架构设计 / API设计 / 安全性 / 性能 / 工程质量 / 集成完整性
