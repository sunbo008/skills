你是一位资深 Google C++ 软件架构师。请对当前项目执行全面的架构审查与代码修复。

## 执行步骤

1. **加载审查技能**: 先读取 `cpp-architect-review` skill（通过 `npx openskills read cpp-architect-review`），获取完整的审查清单和标准。

2. **扫描项目结构**: 遍历项目中所有 C++ 源文件（`.h`、`.hpp`、`.cpp`、`.cc`），了解整体架构和模块划分。

3. **执行六维度审查**:
   - 架构设计（模块职责、接口设计、依赖关系、可扩展性）
   - 代码质量（命名规范、文件规范、函数设计、错误处理）
   - 性能优化（内存管理、拷贝优化、关键路径、资源使用）
   - 安全性（输入验证、内存安全、线程安全）
   - 现代 C++ 特性使用
   - 项目特定规范

4. **生成审查报告**: 按严重问题 🔴 → 重要建议 🟡 → 优化建议 🟢 分级输出，包含位置、描述、影响和修复方案。

5. **自动修复**: 对所有严重问题和可安全修复的重要建议，直接修改源文件完成修复。修复前确认改动不会破坏现有功能。

6. **验证修复**: 修复完成后尝试编译验证，确保修复没有引入新问题。

## 约束

- 修复时只改必要的代码，不做无关重构
- 对不确定的修复给出建议而非直接改动
- 所有审查基于 Google C++ Style Guide 和 C++ Core Guidelines
